# AI-SSH 演进方案：集成 GMSSH 核心可视化与 AI 运维特性

本方案旨在将类 GMSSH 的核心能力（服务器状态监控、Docker 可视化管理、服务启停、AI 智能诊断）无缝集成到当前基于 Tauri + Vue 3 + Rust 的 `AI-SSH` 项目中。

## 一、 核心功能规划与对标

我们在现有的“终端”、“文件浏览器”和“脚本中心”基础上，新增一个 **“服务器概览 (Dashboard)”** 模块，将其作为单台服务器的核心控制台。

### 1. 服务器资源大盘 (System Dashboard)
- **功能**：实时可视化展示 CPU 使用率、内存占用、磁盘空间分布、网络 I/O。
- **UI 设计**：环形图、波浪线图（使用 ECharts 保持专业感），保持现有的暗黑毛玻璃风格。
- **技术实现**：
  - **Rust 后端**：通过现有的 `ssh2` Session，定期（如每 3 秒）后台非阻塞执行获取系统状态的通用 Shell 命令（例如 `top -bn1`，`free -m`，`df -h`）。
  - **数据解析**：在 Rust 端解析命令输出，封装成结构化的 JSON 返回给前端。

### 2. Docker 可视化面板 (Docker Management)
- **功能**：列出所有容器（运行中/已停止），支持启动、停止、重启、删除，以及一键查看容器实时日志。
- **技术实现**：
  - **拉取数据**：通过 SSH 执行 `docker ps -a --format '{{json .}}'` 直接获取 JSON 格式的容器列表。
  - **日志追踪**：利用现有的 PTY 通道，开启一个静默的终端执行 `docker logs -f <container_id>`，并将流式数据推送到前端展示。

### 3. 系统服务管理 (Service Controller)
- **功能**：可视化管理 `systemd` 服务，查看 Nginx、MySQL、Redis 等关键服务的运行状态，支持 `start / stop / restart`。
- **技术实现**：
  - 执行 `systemctl list-units --type=service --state=running,failed` 获取服务列表。

### 4. AI 智能服务器诊断 (AI Diagnostics)
- **功能**：当服务器负载过高、或某服务崩溃时，提供“一键 AI 诊断”按钮。
- **技术实现**：
  - **上下文收集**：Rust 端自动收集最近的系统日志 (`journalctl -n 50`)、高资源占用进程 (`top -bn1`)、或者出错的容器日志。
  - **AI 推理**：组合这些上下文与设定的 Prompt，调用现有的 `DeepSeek API`，让大模型分析根本原因并给出修复建议（甚至直接生成修复脚本，用户点击即可执行）。

## 二、 架构调整与技术栈

### 1. 前端路由与布局调整
目前的布局主要是左侧服务器列表，右侧是功能区。建议为**每一台连接的服务器**建立一个 Tab 页结构，或在侧边栏选中服务器后，提供二级导航：
- `[>_] 终端 (Terminal)`
- `[📊] 概览 (Dashboard)` 🌟 **新增**
- `[🐳] Docker` 🌟 **新增**
- `[📁] 文件 (SFTP)`
- `[📜] 脚本分发 (Ops)`

### 2. Rust 后端指令通道扩展
为了不干扰常规终端的输入输出，我们需要在 Rust 后端为可视化数据请求建立**独立的 SSH Channel (通道)**。
- 终端界面占用一个交互式 PTY Channel。
- 后台数据轮询（CPU、Docker 列表）通过原生的 `Session::channel_session()` 执行一次性 `exec` 命令，快速获取数据后关闭通道。这样不会影响主终端的体验。

## 三、 分阶段实施计划

### 阶段一：数据底座与资源大盘 (1周)
1. 在前端引入图表库 `echarts` 和 `vue-echarts`。
2. 在 Rust 端编写 `get_sys_info` 接口，通过独立的 SSH exec 通道执行监控命令。
3. 实现“服务器概览”面板，展示 CPU、内存、磁盘的动态图表。

### 阶段二：Docker 容器管理引擎 (1周)
1. 编写获取 Docker 信息的 Shell 脚本解析器，对接 Tauri IPC。
2. 开发 Docker 管理前端界面，包含容器卡片、状态标签（绿色运行/红色停止）。
3. 实现基础操作（Start/Stop/Restart）。

### 阶段三：AI 高级智能集成 (1周)
1. 整合现有的 `AiChatBox`，使其能够主动读取当前 Dashboard 上的异常状态（如内存爆满的报警红字）。
2. 开发“一键智能诊断”按钮，定义数据收集 Pipeline -> 组装 Prompt -> 渲染大模型修复建议的完整流。

---

## 四、 核心价值总结
按照此方案进行演进，`AI-SSH` 将从一个“带 AI 辅助的 SSH 客户端”彻底进化为一个**“面向开发者的现代化服务器集群管理中枢”**。它将完全覆盖普通开发者、独立开发者的日常运维需求，体验上直接对标甚至超越现有的宝塔面板、GMSSH 等产品，且保持更极致的轻量级和原生质感。
